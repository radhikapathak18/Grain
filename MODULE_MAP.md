## Critical user journeys (for E2E selection)

1. **Happy path:** Login → Select products → Ask Explore question → Watch stream → Click citation → See evidence → Close → Switch role → Ask same question → Notice different framing.
2. **Empty results:** Verify shape with no-match question → polite empty message bubble + 0 citations.
3. **Cancel-and-retry:** Send → mid-stream press Send again → first stream aborts, second starts.
4. **Rate limit:** Burst >20 requests in 60s → 429 with `Retry-After`.
5. **Concurrency cap:** Two simultaneous streams from one IP → second rejected.
6. **Auth guard chain:** Hit `/chat` unauthenticated → redirect to `/login`. Login but skip product confirm → redirect to `/select`.
7. **Source detail variants:** Open citation for gong / slack / pendo / zoom / confluence → each renders correct layout. Open citation with synthesized placeholder → renders excerpt list, no fake body.
8. **Report view:** Navigate `/report` → all theme cards + emerging issues render with citation chips that open the evidence panel.
9. **Persisted session:** Reload page mid-chat → user + products restored, history NOT restored (by design).
10. **Trends shape recency:** Ask with `trends` shape → claims sorted by `most_recent_evidence_at`, no claims older than 12 months.

## Public API surface (for OpenAPI/contract reference)

```
GET    /api/health                           → { status: 'ok', service: 'grain-api' }
POST   /api/auth/login        body: LoginRequest  → LoginResponse | { error } 4xx
POST   /api/chat/stream       body: ChatRequest   → SSE: status|delta|citation|done|error
GET    /api/claims            ?ids=CL-0001,CL-0002 → { claims: Claim[] }
GET    /api/claims/:id                              → Claim | { error } 404
GET    /api/reports/monthly                         → MonthlyReport
GET    /api/sources/:id                             → SourceDocument | { error } 404
```
